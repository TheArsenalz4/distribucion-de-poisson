# Actividad para realizar:

Los estudiantes deberán desarrollar un programa computacional que permita resolver problemas relacionados con diferentes distribuciones de probabilidad, tales como la distribución normal, binomial, Poisson, etc. El programa debe ser capaz de realizar cálculos precisos, mostrar resultados y manejar diferentes situaciones de entrada.

# Presentación del Proyecto:

Durante la presentación, los equipos deberán demostrar el funcionamiento de su programa al resto del curso. La presentación debe incluir una descripción clara del proyecto, una demostración práctica del programa, destacando las características clave, el enfoque técnico utilizado y el manejo de errores. Se espera que los estudiantes respondan preguntas del público al final de la presentación.

# Pauta de Evaluación del Proyecto:

## Aspectos Para Evaluar:

1. **Precisión y Exactitud (20 puntos)**
    - Precisión de los cálculos para las distribuciones de probabilidad seleccionadas.
    - Coincidencia de resultados con las teorías y fórmulas establecidas para cada distribución.

2. **Interfaz y Usabilidad (20 puntos)**
    - Claridad y facilidad de uso de la interfaz del programa.
    - Presentación clara y comprensible de los resultados sin utilizar gráficos.

3. **Originalidad y Complejidad del Código (20 puntos)**
    - Enfoque original para resolver problemas de distribuciones de probabilidad.
    - Complejidad del código y eficiencia en la resolución de problemas planteados.

4. **Presentación (20 puntos)**
    - Habilidad para comunicar de manera clara y concisa los aspectos básicos del proyecto.
    - Demostración efectiva de la funcionalidad del programa, destacando sus características principales.
    - Habilidad para responder preguntas o dudas de manera adecuada durante la presentación.

5. **Manejo de Errores y Robustez (15 puntos)**
    - Capacidad del programa para manejar entradas incorrectas o situaciones de error.
    - Robustez del programa para producir resultados confiables en diversas condiciones.

6. **Documentación y Soporte (5 puntos)**
    - Calidad y claridad en la documentación presentada durante la exposición.

# Presentación de ejemplo del Proyecto:

- **Introducción:** Saludo y presentación del equipo.
- **Descripción del Proyecto:** Explicación del objetivo del programa y su importancia en ingeniería.
- **Demostración:** Ejemplo práctico de resolución de un problema de distribución de probabilidad.
- **Enfoque Técnico:** Detalles sobre el enfoque utilizado en el desarrollo del programa.
- **Manejo de Errores y Robustez:** Demostración de cómo el programa gestiona situaciones de error.
- **Conclusión:** Resumen de los puntos clave y agradecimientos.
